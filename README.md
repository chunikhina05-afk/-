# 🧠 Хранитель знаний

ИИ-агент в Telegram, который превращает поток ваших сообщений в **структурированную личную базу знаний**: извлекает сущности и связи, строит граф, ищет по смыслу и отвечает на вопросы по вашим заметкам.

> Пишешь как думаешь — получаешь структурированную базу знаний.

---

## Возможности

- 📥 **Захват** — любое сообщение в Telegram сохраняется мгновенно.
- 🕸️ **Граф знаний** — автоматически извлекает сущности (9 типов) и связи (7 типов).
- 🔍 **Поиск по смыслу** — векторные эмбеддинги (768), а не ключевые слова.
- 💬 **RAG-ответы** — отвечает строго по вашим данным; если ответа нет — честно говорит.
- 🕰️ **Память** — история изменений сущностей и контекст диалога; уточняющие вопросы.
- 🏷️ **Теги** — автоматические теги и поиск по ним.

**Типы сущностей:** `Person`, `Company`, `Project`, `Task`, `Idea`, `Technology`, `Book`, `Event`, `Link`
**Типы связей:** `WORKS_AT`, `KNOWS`, `PARTICIPATED_IN`, `RELATED_TO`, `CREATED`, `TASK_FOR`, `REFERENCES`

---

## Команды

| Команда | Действие |
|---|---|
| `(любой текст)` | Сохранить заметку и автоматически разобрать её |
| `?вопрос?` (`?` в начале или конце) | Ответ по базе знаний (RAG) |
| `/graph <имя>` | Связи сущности — текстом |
| `/визуал` · `/картинка` · `/picture` | Интерактивный граф всей базы (HTML-файл) |
| `/история <имя>` · `/history <имя>` | Хронология изменений сущности |
| `/тег <тег>` · `/tag <тег>` | Заметки по выбранному тегу |

## Как это работает

**Сохранение заметки:**
```
Сообщение → запись в БД → эмбеддинг (768) → извлечение сущностей и связей (LLM)
          → запись в граф + теги → подтверждение
```

**Ответ на вопрос (RAG):**
```
Вопрос → эмбеддинг → семантический поиск (топ-5 заметок) + соседи по графу
       → сборка контекста → ответ Gemini по-русски
```

## Технологии

- **[python-telegram-bot](https://python-telegram-bot.org/)** — интерфейс Telegram (async).
- **[Google Gemini](https://ai.google.dev/)** — `gemini-2.5-flash` (извлечение + генерация), `gemini-embedding-001` (эмбеддинги, 768 измерений).
- **PostgreSQL + [pgvector](https://github.com/pgvector/pgvector)** — заметки, граф и векторный поиск (например, на [Neon](https://neon.tech/)).
- **[pyvis](https://pyvis.readthedocs.io/)** — интерактивная визуализация графа.

---

## Требования

- Python **3.10+**
- PostgreSQL с расширением **`pgvector`** (создаётся автоматически при старте: `CREATE EXTENSION IF NOT EXISTS vector`)
- Доступ к **Google Gemini API**

## Настройка

> ⚠️ **Не храните токены и ключи прямо в коде.** Используйте переменные окружения (или файл `.env`).

```bash
export TELEGRAM_TOKEN="ваш_токен_бота"
export DATABASE_URL="postgresql://user:password@host/db?sslmode=require"
export GEMINI_API_KEY="ваш_ключ_gemini"
```

В коде читайте их так:

```python
import os

TOKEN        = os.environ["TELEGRAM_TOKEN"]
DATABASE_URL = os.environ["DATABASE_URL"]
GEMINI_API_KEY = os.environ["GEMINI_API_KEY"]
```

## Запуск

```bash
python bot.py
```

При первом старте автоматически создаются таблицы:
`notes`, `entities`, `relations`, `entity_history`.

---

## Структура базы данных

| Таблица | Назначение |
|---|---|
| `notes` | Заметки + эмбеддинги (`vector(768)`) + теги |
| `entities` | Сущности (тип, имя, описание); уникальность по `(telegram_id, type, name)` |
| `relations` | Связи между сущностями |
| `entity_history` | История изменений по каждой сущности |
